# chat — product notes

## Concept

A chat-native workspace for OpenCloud-style workflows.

The UI begins with a single project chat page and assumes multi-agent participation by default. Files and memory are not treated as separate destinations; they expand contextually alongside the active conversation.

## Demo goals

- Show three agents with distinct OKLCH accent systems
- Make the main chat feel like a real working room instead of a generic messenger
- Reveal file activity only when the conversation touches files
- Reveal memory only when agreements are promoted or updated

## Primary layout

- Left: project + thread navigation
- Center: multi-agent project chat
- Right: contextual panel for files or memory

## Demo story

Owner asks Mona to shape a new chat product demo.
Mona delegates UI structure to Rune and memory semantics to Iris.
The conversation references files and promotes agreements into project memory.
